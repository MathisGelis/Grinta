### **BETA TEST PLAN – GRINTA**

## **1. Project context**
Grinta is a fitness tracking and social application designed for gym users of all levels.
The goal of the platform is to allow users to track their workout sessions, monitor their progress over time, and share their training activity with others in order to benefit from the motivational and social effects generated by collective practice.

## **2. User role**

The following role will be involved in beta testing:

| **Role Name** | **Description** |
|--------------|------------------|
| Everyone | Any gym user using the application to track and share workout activity |

---

## **3. Feature table**

The following features will be shown during the defense:

| **Feature ID** | **User role** | **Feature name** | **Short description** |
|--------------|---------------|------------------|------------------------|
| F1 | Everyone | User authentication | Create an account and securely log in to the application |
| F2 | Everyone | Exercise creation | Create and manage custom exercises |
| F3 | Everyone | Split creation | Create workout splits composed of multiple exercises |
| F4 | Everyone | Workout session creation | Log a workout session based on a split or free exercises |
| F5 | Everyone | Workout history | View past workout sessions and personal training history |
| F6 | Everyone | Session sharing | Share completed workout sessions with the community |
| F7 | Everyone | Social feed | View shared workout sessions from other users |
| F8 | Everyone | Friend connections | Connect with other users and manage a friends list |
| F9 | Everyone | Social interactions | Like or react to friends’ shared workout sessions |
| F10 | Everyone | Profile & progress | View personal profile and track performance progression |

---

## **4. Success Criteria**

The beta version will be considered successful if the following criteria are met:

| **Feature ID** | **Key success criteria** | **Indicator / metric** | **Result** |
|--------------|--------------------------|-------------------------|------------|
| F1 | Users can authenticate without issues | 20 attempts, 0 failures related to the application | Achieved (20/20) |
| F2 | Users can create and reuse exercises | 20 creations, 0 failures related to the application | Achieved (20/20) |
| F3 | Splits can be created and edited correctly | 20 creations and editions, 0 failures related to the application | Achieved (20/20) |
| F4 | Workout sessions are saved accurately | 20 saves, 0 failures related to the application | Achieved (20/20) |
| F5 | Users can access their workout history | History load time < 2s | Achieved |
| F6 | Sessions can be shared successfully | 20 shares, 0 failures related to the application | Achieved (20/20) |
| F7 | Social feed displays relevant content | Feed refresh without errors | Achieved |
| F8 | Users can add and manage friends | Friend request acceptance flow validated | Achieved |
| F9 | Social interactions work as expected | Like/reaction actions recorded correctly | Achieved |
| F10 | Progress tracking reflects real data | Progress metrics consistency verified | Achieved |
